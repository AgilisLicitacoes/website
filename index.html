O Problema

Editais são complexos.
A análise manual, cara.

Milhares de licitações abertas por mês, documentos extensos sem padronização, linguagem jurídica densa. O setor farmacêutico perde tempo e dinheiro.

⏱️

Análise Lenta e Trabalhosa

Editais chegam a centenas de páginas. A leitura manual consome horas de profissionais especializados em cada oportunidade.

📋

Sem Padronização entre Órgãos

Cada prefeitura, hospital ou secretaria organiza seu edital de forma diferente. Algoritmos simples não conseguem lidar com essa variabilidade.

❌

Erros Humanos Custam Caro

Perder um prazo ou requisito obrigatório pode significar a desclassificação imediata da proposta — e a perda do contrato.

🏢

Acesso Desigual ao Mercado

Empresas menores não conseguem acompanhar o volume de licitações disponíveis. O mercado público fica concentrado em poucos grandes players.

Como Funciona

Pipeline inteligente de análise

Múltiplos algoritmos atuam em sequência, refinando progressivamente os resultados para máxima precisão.

1
Upload & Processamento

Envio e Preparação do Edital

O usuário faz upload do PDF ou arquivo de texto. O sistema extrai o conteúdo bruto, realiza limpeza de texto, normalização de caracteres e segmentação em seções. Um hash é gerado para evitar reprocessamento de arquivos duplicados.

PDF Parser Regex Hashing Python
2
Lógica & Heurísticas

Extração por Regras e Padrões

Algoritmos de lógica e expressões regulares identificam padrões conhecidos: datas, valores, códigos de produtos, UASG, prazos de entrega e documentos exigidos. Essa camada trata os casos mais frequentes com alta velocidade.

Regex Rule Engine NLP Clássico
3
Machine Learning

Modelos de Linguagem Especializados

Para trechos ambíguos ou estruturas atípicas, entram modelos de aprendizado de máquina — incluindo variantes do BERT ajustadas para texto jurídico e farmacêutico. Esses modelos refinam e validam as informações extraídas na etapa anterior.

BERT Legal-BERT Fine-tuning FastAPI
4
Resultado

Relatório Estruturado para o Usuário

O sistema consolida as informações extraídas em um relatório organizado: prazos, itens licitados, critérios de habilitação, documentos obrigatórios e alertas de risco. Tudo salvo no histórico e acessível a qualquer momento.

React PostgreSQL Node.js

Tecnologia

Arquitetura
moderna e robusta

Módulos independentes garantem escalabilidade, manutenibilidade e alta performance em cada camada do sistema.

Frontend — React + shadcn/ui

Interface responsiva e moderna com componentes reutilizáveis. Experiência fluida para upload, acompanhamento da análise e visualização dos resultados.

Backend — Node.js + Express

API principal assíncrona para gerenciamento de usuários, autenticação JWT, controle de uploads e comunicação com o banco de dados.

Módulo IA — Python + FastAPI

Pipeline de processamento de linguagem natural com pandas, modelos BERT e algoritmos de ML especializados em texto jurídico.

Banco de Dados — PostgreSQL

Armazenamento estruturado e confiável de usuários, análises e resultados extraídos, com suporte a consultas complexas.
